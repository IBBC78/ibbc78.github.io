Backend Architecture

User Authentification System

On the backend, Marvel Travel relies on PHP to handle core business logic and data management. User authentication is managed through PHP sessions, allowing users to register, log in, and maintain their session securely across pages. User credentials are stored in a JSON file (utilisateurs.json), which acts as a simple database substitute for this academic project.


Data persistence is achieved by reading and writing JSON files. The application maintains three main JSON data stores: voyages.json for destination information, commandes.json for reservation records, and utilisateurs.json for user accounts. When a user makes a reservation, the backend updates commandes.json to store this new booking, associating it with the logged-in user.

Booking & Payement

The payment process is simulated through an API provided by the school. This integration validates payment details such as card number and expiration date, mimicking a real-world transaction. Upon successful payment, the system generates a confirmation receipt, typically in PDF format, providing users with proof of their booking.


When a user makes a reservation, the application checks the availability of the selected destination and updates the commandes.json file accordingly. This ensures that users can only book available trips, preventing overbooking.

User profile management is another key backend feature. Each user has access to a personalized profile page where they can view their reservation history, which is filtered from the commandes.json file based on their username. Users can also customize their avatars, with image uploads handled locally on the server. Persistent sessions ensure that users remain logged in as they navigate through the site.

All users receive a fictitious passport on their profile.
